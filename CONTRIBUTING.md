# Contributing

Verion is a product about confidence.

It helps developers verify AI-generated software before they ship. The product should reduce uncertainty, identify meaningful issues, explain likely causes, and prepare the next action.

Contributing to Verion means protecting that focus.

## What Verion Is

Verion is the verification layer after AI code generation.

It is a calm product reviewer. It explores the application, finds what matters, groups symptoms into useful diagnoses, and helps Codex fix issues with the right context.

## What Problems It Solves

- Developers do not know whether AI-generated code is safe to ship.
- Manual review is slow and inconsistent.
- Generic warnings create noise instead of confidence.
- AI fixes need better context to be useful.
- Shipping should feel like a decision, not a guess.

## How to Evaluate New Features

Ask these questions before adding anything:

- Does this help the non-negotiable MVP loop?
- Does this increase shipping confidence?
- Does this reduce developer anxiety?
- Does this make the next action clearer?
- Does this improve judgment, or only add more information?
- Would the demo be weaker without it?

If the answer is no, the feature belongs in `NOT_BUILDING.md`.

## How to Reject Unnecessary Complexity

Prefer the smallest product surface that proves the loop.

Reject features that require explanation before they create value. Reject features that make Verion feel like a generic dashboard. Reject features that add options before the core opinion is strong.

A good Verion feature should make the developer feel more certain about what happened and what to do next.

## Contribution Standard

Protect clarity. Protect trust. Protect the loop.

The best contribution is not the largest one. It is the one that makes Verion feel more inevitable.
