# Demo

## Three-Minute Flow

### Opening Hook

AI can generate code in seconds. The hard part is knowing whether you can ship it.

### Problem

Developers are using AI coding agents to move faster, but every generated change creates a new question: what did it miss?

Manual checking is slow. Traditional QA tools require setup. Generic warnings create more uncertainty. Verion is built for the moment after the AI writes the code and before the developer ships.

### Live Coding With Codex

Show Codex generating or changing a small product flow.

The feature appears to work. Click through the obvious path once. The audience should see why a developer might be tempted to ship.

### Verification

Open Verion and press Verify.

Verion autonomously explores the application. The key moment is that the developer does not write a manual checklist. Verion starts checking the product behavior directly.

### Issue Discovery

Verion finds a meaningful issue in the generated experience.

Do not frame it as a generic bug list. Show that Verion discovered something a developer could realistically miss when trusting an AI-generated change.

### Root Cause Reasoning

Verion groups the observed symptoms by likely root cause.

The product should make the issue feel understandable, not merely detected. The audience should see a clear explanation of what matters and why.

### Auto-Fix

Send the prepared context to Codex.

Codex fixes the issue using the focused context from Verion. The handoff should feel like a natural continuation of the verification loop, not a separate workflow.

### Verification Again

Run Verion again.

Show that the original issue no longer appears. The second pass is the confidence moment.

### Ready to Ship

End on the ready-to-ship state.

The developer did not just generate code. They verified the product, fixed the issue, and confirmed the outcome.

## Closing Sentence

AI writes the code. Verion gives you the confidence to ship it.
